Highcharts JS For Dummies

This post will walk you through Highcharts JS with basic Information and Implementation

Information

The following provides a foundational overview and useful information about Highcharts JS and JavaScript libraries in general.

JS libraries

As W3Schools puts it, "JavaScript is the programming language of the web" that "all modern HTML pages are using." It "is one of three languages all web developers must learn," including HTML and CSS. When developing a web page with dynamic content, JavaScript is the primary programming language used to add functionality and behavior, while HTML is the primary markup language to add structure and definition and CSS is the primary stylization language to add layout and design. Being that JavaScript is an extremely successful, free and open source language, many users have created JavaScript libraries which allow for easier development and additional features. According to Similar Tech's JavaScript statistics, jQuery, the most popular JavaScript library in use today, is used by over 80% of the most visit websites.

Highcharts JS

Highcharts is a JavaScript library that aids developers in the creation of data visualization through charts. And as the tagline goes on their homepage, Highcharts can "make your data come alive." "Highcharts supports line, spline, area, area-spline, column, bar, pie, scatter, angular gauges, area-range, area-spline-range, column-range, and polar charts." Highcharts boasts that it is "used by tens of thousands of developers and 59 out of the world's 100 largest companies."

Highsoft AS

The company behind Highcharts JS is known as Highsoft AS, and according to Similar Tech's JavaScript statistics, Highsoft AS has three JS libraries in the top JavaScript technologies most used on the web today, including Highslide at #6, Highcharts at #8, and Highstock at #11. At the top, jQuery is #1, and as you'll find out next, Highcharts JS requires a framework like jQuery for installation.

Implementation

The following are step by step instructions for implementing Highcharts JS into a webpage. The example will create a Highcharts bar chart that uses the data from Similar Tech's statistics on today's leading Javascript technologies.

To follow along with the implementation, check out the Highcharts JS documentation.

Installation

To implement Highcharts JS into your web page, you need to import two files.

1) You'll need a JS framework like jQuery, MooTools or Prototype each of which is a widely-used JavaScript library designed to simplify JavaScript programming (another alternative is the Highcharts Standalone Framework). This example will use jQuery: http://code.jquery.com/jquery-2.1.1.min.js

2) You'll need the Highcharts JS file itself: http://code.highcharts.com/highcharts.js

You have two options for importing the JS files.

Option #1: Their server

You can simply import the two files by referencing directly from the respective servers' CDN:

<script src="http://code.jquery.com/jquery-2.1.1.min.js"></script>
<script src="http://code.highcharts.com/highcharts.js"></script>

Option #2: Your server

You can import the two files by referencing them from a folder on your server. For example, you can create a folder called "js" where you can download both files. Simply open the links above, right-click on both pages in your browser, and select the option, "Save Asâ€¦" to put them in your "js" folder.

<script src="./js/jquery-2.1.1.min.js"></script>
<script src="./js/highcharts.js"></script>

Demos

The Highcharts JS website features some good demos that have buttons below each example, "View Options" and "Edit in JSFiddle," which allow you to see and test the code respectively. The demos page is also a great start to figure out which chart would be best suitable for the data you wish to visualize.

API

Once you've looked at the demos and gotten a feel for the chart options and visual themes, take a look at the API. Here you can delve down deeper into configuration options for the initial chart setup. For instance, if you click on the "chart: {}" under "$("#container").highcharts({}," you'll see the general chart options like "backgroundColor" and "type." You'll also notice that the carrot drop down has reveal all the options in the left navigation area, which also shows you the default value for each option.

HTML

Now that you have glanced at the documentation, you can start writing the necessary HTML code in the "<body>" of your webpage. Highcharts JS allows you to create a chart within a "<div>" element with a "container" id attribute. Using the "style" attribute or external CSS, you'll want to specify things such as the width, height or margin.

<body>
	<div id="container" style="min-width: 400px; width: 95%; height: 700px; margin: 0 auto;"></div>
</body>

JS

Next, you'll want to write some JavaScript in the <head> of your webpage. Use jQuery to create an anonymous function that references the chart's id and calls the "highcharts()" function from Highcharts JS library using the dot operator.

<script type="text/javascript">
	$(function () {
		$('#container').highcharts({
		// next bits of code go here!
		});
	});
</script>

You're gonna wanna look at the API to write more JavaScript where the comments say, "// next bits of code go here!" The first item you looked at earlier was "chart: {}" In here the most important item for you will likely by "type." This example uses "type: 'bar'". Just as the API sidebar exemplifies, implementing this code would look like so:

...
$('#container').highcharts({
	chart: {
		type: 'bar'
	},
	// more code yet!
});
...

Next, title the chart, "Top JavaScript Technologies," and to give credit to Similar Tech, add a subtitle specifying the source.

...
title: {
text: 'Top JavaScript Technologies'
},
subtitle: {
	text: 'Source: SimilarTech.com'
},
...

In Highcharts, the tooltip is enabled by default, and because Similar Tech displays percentages in its charts, use the suffix % on the tooltip.

...
tooltip: {
	valueSuffix: '%'
},
...

And now for the axes! On the x-axis, put each of the top 8 JS technologies into a category.

...
xAxis: {
	categories: ['jQuery', 'jQuery UI', 'Errorception', 'Bootstrap', 'Modernizr', 'Highslide', 'Backbone.js', 'Highcharts']
},
...

On the y-axis, you'll have the percentages, so title it as such.

...
yAxis: {
	title: {
		text: 'Percentage of websites'
	}
},
...

Finally, Highcharts uses the keyword "series" to allow you to set the data for each category. Also, Similar Tech uses statistics for the "Top 1M," "Top 100K" and "10K" websites, so you'll want to input the data for each of these.

...
series: [{
	name: 'Top 1M',
	data: [58.69, 15.99, 1.10, 3.17, 2.74, 0.94, 0.21, 0.20]
}, {
	name: 'Top 100K',
	data: [61.27, 19.94, 4.04, 3.48, 3.58, 0.80, 0.34, 0.36]
}, {
	name: 'Top 10K',
	data: [60.20, 21.01, 8.72, 2.99, 4.31, 0.37, 0.55, 0.64]
}]
...
